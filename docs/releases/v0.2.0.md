# OrgScript v0.2.0

OrgScript v0.2.0 is the first release built on a real parsing pipeline instead of line-based validation.

This release introduces a proper lexer, parser, AST layer, semantic validation, and canonical JSON export. It marks the transition from a specification-first project to an actual language implementation.

## Highlights

- Real v0.2 language pipeline
- lexer
- parser
- AST
- semantic validation
- Canonical JSON export
- Clear AST modeling for core language constructs
- English-first hero example
- GitHub project basics added for public release readiness

## Changed

- `validate` now runs on parsed structure instead of line-based checks
- CLI now supports:
- `validate`
- `export json`
- `when` and `if` remain separate in the AST by design
- `require` is modeled as its own semantic node
- `policy when ... then ...` is modeled as a clause, not as a loose `ThenNode`
- The hero example is now fully English:
- `examples/craft-business-lead-to-order.orgs`

## Project status

OrgScript is not a programming language.
It is a human-readable, AI-interpretable description language for business logic, processes, rules, roles, policies, events, and operational systems.

v0.2.0 establishes the first real implementation baseline.

## Notes

This release focuses on structural correctness and canonical representation.
Formatter and linting are intentionally deferred until AST and JSON output are locked down by golden tests.

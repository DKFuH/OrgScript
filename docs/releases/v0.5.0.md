# OrgScript v0.5.0

OrgScript v0.5.0 is a developer-experience, adoption, and polish release.

This release does not expand the language itself. Instead, it improves how OrgScript is read, tried, understood, and used in practice.

## Highlights

### Better front door and onboarding
- The README now presents the core value of OrgScript more clearly
- Added a stronger quickstart path
- The “three-value flow” is now visible near the top:
  - OrgScript source
  - Mermaid diagram
  - Markdown summary

### Improved VS Code highlighting
- Refined the initial VS Code syntax grammar
- Better handling for inline keywords and dotted references
- Improved local editor onboarding documentation

### Curated example catalog
- The example catalog is now structured into:
  - simple
  - realistic
  - advanced
- Each example is easier to understand as a learning and adoption asset

### Diagnostics UX polish
- CLI text output is now more consistent and deliberate
- `ok` is now reported as `passed`
- Diagnostic codes are shown more clearly in bracketed form
- Output spacing and readability were improved without changing the documented JSON contract

### Language evolution guidance
- Added `docs/language-evolution.md`
- Clarifies what belongs to core language evolution vs tooling, exporters, and editor integrations
- Establishes a cleaner baseline for future breaking-change discussions

## Scope note

v0.5.0 intentionally focuses on usability and adoption.

It does **not** introduce:
- new language keywords
- new block types
- parser redesigns
- IntelliSense / hovers / semantic tokens
- speculative future syntax

## Verification

Verified on the current release state with:

- `npm test`
- `npm run check:all`
- `node ./bin/orgscript.js check ./examples/craft-business-lead-to-order.orgs`
- `node ./bin/orgscript.js export mermaid ./examples/order-approval.orgs`
- `node ./bin/orgscript.js export markdown ./examples/order-approval.orgs`

## Project direction

OrgScript is becoming easier to try, easier to demo, and easier to trust.

The next release cycle will focus on integrations, generated artifacts, and stronger showcase value.
